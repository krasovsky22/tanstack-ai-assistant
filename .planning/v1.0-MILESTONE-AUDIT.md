### MEM-05 Gap Detail

The Phase 02 verifier confirmed 7 `void indexDocument` call sites — all of which were correct *for the HTTP write path*. However, the integration check found that Phase 01's `runIngestion()` creates jobs via a direct `db.insert(jobs)` in `src/services/mail-ingestion.ts` (lines 184–196), bypassing the POST `/api/jobs` HTTP handler entirely. The `indexDocument` hook lives only in the HTTP handler (`src/routes/api/jobs/index.tsx` lines 60–71). Result: jobs auto-created through email ingestion are never indexed in `memory_jobs`.

**Fix required:** Add `void indexDocument(...)` in `mail-ingestion.ts` after the `db.insert(jobs).returning()` call, mirroring the pattern in `api/jobs/index.tsx`.
