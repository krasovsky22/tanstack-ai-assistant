    1. Private jiraFetch helper (NOT exported):
    ```typescript
    function jiraFetch(baseUrl: string, token: string, path: string, options?: RequestInit) {
      const url = `${baseUrl.replace(/\/$/, '')}/rest/api/2${path}`;
      return fetch(url, {
        ...options,
        headers: {
          'Authorization': `Bearer ${token}`,
          'Content-Type': 'application/json',
          'Accept': 'application/json',
          ...(options?.headers ?? {}),
        },
      });
    }
    ```

    2. Env var guard helper (inline in each handler — do not share state at module level):
    Read JIRA_BASE_URL and JIRA_PAT at the start of each .server() handler.

    3. Tool: jira_search
    - inputSchema: { jql: z.string(), maxResults: z.number().int().min(1).max(50).optional().default(10), startAt: z.number().int().min(0).optional().default(0) }
    - GET /search with URLSearchParams: jql, maxResults, startAt, fields='summary,status,assignee,description,priority,issuetype,created,updated'
    - Returns: { total: data.total, maxResults: data.maxResults, startAt: data.startAt, issues: data.issues.map(i => ({ key: i.key, id: i.id, fields: i.fields })) }

    4. Tool: jira_get_issue
    - inputSchema: { issueKey: z.string().describe('Jira issue key, e.g. PROJ-123') }
    - GET /issue/{issueKey}?fields=summary,status,assignee,description,priority,issuetype,created,updated,comment
    - Returns: { key: data.key, id: data.id, fields: data.fields }

    5. Tool: jira_update_description
    - inputSchema: { issueKey: z.string(), description: z.string().describe('New plain-text description') }
    - PUT /issue/{issueKey} with body JSON.stringify({ fields: { description } })
    - On res.ok (204): return { success: true, issueKey }
    - DO NOT call res.json() — 204 has no body

    6. Tool: jira_add_comment
    - inputSchema: { issueKey: z.string(), comment: z.string().describe('Comment text to add') }
    - POST /issue/{issueKey}/comment with body JSON.stringify({ body: comment })
    - Returns: { success: true, comment: await res.json() } — 201 has a body

    7. Tool: jira_get_comments
    - inputSchema: { issueKey: z.string() }
    - GET /issue/{issueKey}/comment
    - Returns: { issueKey, comments: data.comments.map(c => ({ id: c.id, author: c.author?.displayName ?? c.author?.name, body: c.body, created: c.created })) }

    8. Tool: jira_assign_issue
    - inputSchema: { issueKey: z.string(), username: z.string().describe('Jira username (name field) to assign. Use null string "null" to unassign.') }
    - PUT /issue/{issueKey}/assignee with body JSON.stringify({ name: username === 'null' ? null : username })
    - On res.ok (204): return { success: true, issueKey, assignedTo: username }
    - DO NOT call res.json() — 204 has no body

    Error handling for all tools (wrap entire handler in try/catch):
    ```typescript
    try {
      const res = await jiraFetch(baseUrl, token, path, options);
      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        return { success: false, error: (err.errorMessages ?? []).join('; ') || `Jira returned status ${res.status}` };
      }
      // success path
    } catch (err: any) {
      return { success: false, error: err.message ?? 'Unknown error' };
    }
    ```

    Do NOT use any external Jira SDK or axios. Use native fetch only.
    Do NOT use accountId for assignee — use name (Jira Server).
    Do NOT use ADF format — plain strings only.
