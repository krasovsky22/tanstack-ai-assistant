  <files>docker-compose.yml, .env.example</files>
  <action>
Edit docker-compose.yml to add the `elasticsearch` service and wire ELASTICSEARCH_URL across all worker services:

1. Add `elasticsearch` service before the `volumes:` section:
```yaml
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.19.12
    restart: unless-stopped
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - xpack.security.enrollment.enabled=false
      - ES_JAVA_OPTS=-Xms512m -Xmx512m
    ports:
      - "9200:9200"
    volumes:
      - elasticsearch_data:/usr/share/elasticsearch/data
    sysctls:
      - vm.max_map_count=262144
```

2. Add `elasticsearch_data:` to the `volumes:` section (alongside `postgres_data:`).

3. Add `ELASTICSEARCH_URL: http://elasticsearch:9200` to the `environment:` section of ALL four services: `ui`, `gateway`, `jobs`, `cron`. Also add `depends_on: elasticsearch` to `ui`, `gateway`, `jobs`, `cron` services (append to existing `depends_on` list).

4. Edit .env.example — add after the existing DATABASE_URL entry:
```
# Elasticsearch URL (internal Docker hostname when running in Docker, localhost:9200 for local dev)
ELASTICSEARCH_URL=http://localhost:9200
```

5. Also add to .env.example an optional note about WSL2 vm.max_map_count: document that if the elasticsearch container exits immediately, run `sudo sysctl -w vm.max_map_count=262144` on the WSL2 host.

Do NOT add ELASTICSEARCH_URL to the `elasticsearch` service environment — it is the server itself.
