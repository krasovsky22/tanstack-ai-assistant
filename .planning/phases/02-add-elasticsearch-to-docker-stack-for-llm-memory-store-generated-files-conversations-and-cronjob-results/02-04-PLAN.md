<objective>
Wire the four write hooks that feed data into Elasticsearch: conversation saves, job creation, cronjob result logging, and resume generation. All hooks use fire-and-forget (void + error swallowing) so ES failures never break the Postgres write path.

Purpose: Without write hooks, the search_memory tool would always return empty results. This plan connects the data pipeline so newly created records flow into ES indices.
Output: Four modified files with indexDocument() calls after their respective Postgres writes.
</objective>

**src/services/chat.ts — add ES indexing to saveConversationToDb and appendMessagesToConversation:**

Both functions use dynamic imports. Add `indexDocument` call after each Postgres write using the same dynamic import pattern.

In `saveConversationToDb()`, after the `await db.insert(conversations).values(...)` call and the messages insert, add:
```typescript
  // Index into Elasticsearch (fire-and-forget — failure must not break Postgres write)
  const { indexDocument } = await import('@/services/elasticsearch');
  const snippet = messages
    .flatMap((m) => (m.parts as Array<{ type: string; text?: string }>))
    .filter((p) => p.type === 'text')
    .map((p) => p.text ?? '')
    .join(' ')
    .slice(0, 2000);
  void indexDocument('memory_conversations', conversationId, {
    conversationId,
    title,
    source,
    messageSnippet: snippet,
    source_type: 'conversation',
    createdAt: new Date().toISOString(),
  });
```

In `appendMessagesToConversation()`, after the messages insert, add:
```typescript
  // Re-index conversation with updated message snippet
  const { indexDocument } = await import('@/services/elasticsearch');
  const snippet = messages
    .flatMap((m) => (m.parts as Array<{ type: string; text?: string }>))
    .filter((p) => p.type === 'text')
    .map((p) => p.text ?? '')
    .join(' ')
    .slice(0, 2000);
  void indexDocument('memory_conversations', conversationId, {
    conversationId,
    title: conversationId,     // title not available in append context; use ID as fallback
    source: null,
    messageSnippet: snippet,
    source_type: 'conversation',
    createdAt: new Date().toISOString(),
  });
```

Note: `void` is used intentionally — fire-and-forget. Do NOT await the indexDocument call in the critical path.
