    State machine: use a single `type SubmitState = 'form' | 'submitting' | 'success' | 'error'` local state variable (`useState<SubmitState>('form')`). Also track `ticketInfo: TicketInfo | null` and `errorMessage: string`.

    TanStack Form setup (per RESEARCH.md Pattern 2):
    ```typescript
    import { useForm } from '@tanstack/react-form';
    const form = useForm({
      defaultValues: { title: '', description: '' },
      onSubmit: async ({ value }) => {
        setSubmitState('submitting');
        const pageUrl = window.location.href;  // captured at submit time, not render
        const prompt = buildReportPrompt(value.title, value.description, pageUrl);
        const res = await fetch('/api/chat-sync', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            userId: user?.userId ?? null,   // REQUIRED — omitting breaks Jira config
            messages: [{ role: 'user', content: prompt }],
            // NO chatId — non-gateway branch only
          }),
        });
        const data = await res.json();
        if (!res.ok || data.error) throw new Error(data.error ?? 'Request failed');
        const parsed = parseTicketResponse(data.text);
        if (!parsed.success) throw new Error(parsed.error);
        setTicketInfo({ category: parsed.category, ticketKey: parsed.ticketKey, ticketUrl: parsed.ticketUrl });
        setSubmitState('success');
      },
    });
    // Wrap onSubmit in try/catch at the form level; on catch: setSubmitState('error'); setErrorMessage(err.message)
    ```

    Three render states inside AppModal:
    1. `form`: Chakra VStack with two fields using `form.Field`:
       - Title field: `<Input placeholder="Brief summary..." />` with min-length validation (≥3 chars), show field error below input
       - Description field: `<Textarea placeholder="What happened? What did you expect?" rows={4} />` required, show field error below textarea
       - Footer: Cancel button (variant="ghost", calls handleClose) + Submit button (variant="solid", type="submit")
       - Submit button uses `form.Subscribe selector={(s) => s.isSubmitting}` to show `<Spinner size="xs" />` and disable while submitting
    2. `submitting`: Center-aligned `<Spinner />` + "Classifying and creating ticket..." text (no footer needed — user cannot interact)
    3. `success`: VStack with CheckCircle2 icon (green), category label e.g. "Bug report created", `<Link href={ticketInfo.ticketUrl} target="_blank">View ticket {ticketInfo.ticketKey} <ExternalLink size={12} /></Link>`, and a "Close" button in footer
    4. `error`: AlertCircle icon (red) + errorMessage text + "Try Again" button (resets submitState to 'form', preserves form values — do NOT call form.reset())

    handleClose function: resets form (`form.reset()`), sets submitState back to 'form', clears ticketInfo and errorMessage, then calls `onClose()`.
    Do NOT call handleClose when in 'submitting' state (disable the close trigger).

    Icons from lucide-react: CheckCircle2, ExternalLink, AlertCircle, Spinner is from Chakra.
    Use AppModal as the shell: `import { AppModal } from '@/components/AppModal'`.
