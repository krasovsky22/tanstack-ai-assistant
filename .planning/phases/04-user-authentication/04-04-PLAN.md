  <what-built>
    Complete user authentication system:
    - users table in DB with bcrypt-hashed passwords
    - Cookie-based server-side sessions via useSession (TanStack Start)
    - /login page (no register/forgot links, no IconRail/sidebar)
    - All existing pages moved under _protected pathless layout — URLs unchanged
    - Unauthenticated access redirects to /login
    - /api/jobs, /api/cronjobs, /api/conversations filter by session userId
    - /api/chat reads userId from session
    - pnpm create-user CLI script for admin user creation
  </what-built>
  <how-to-verify>
    Prerequisites: Ensure SESSION_SECRET is set in .env (>= 32 chars). Run `pnpm dev`.

    **Test 1 — Unauthenticated redirect:**
    1. Clear all cookies for localhost:3000 in browser DevTools (Application > Cookies > Clear)
    2. Navigate to http://localhost:3000/jobs
    3. Expected: redirected to http://localhost:3000/login
    4. Expected: IconRail and ChatSidebar are NOT visible on the login page

    **Test 2 — Invalid credentials:**
    1. On /login, enter username "wronguser" and password "wrongpass"
    2. Click Sign in
    3. Expected: error message "Invalid credentials" appears, no redirect

    **Test 3 — Valid login flow:**
    1. Run `pnpm create-user testuser mypassword123` in terminal (if not already done)
    2. On /login, enter username "testuser" and password "mypassword123"
    3. Click Sign in
    4. Expected: redirected to http://localhost:3000/ (home page)
    5. Expected: IconRail is visible

    **Test 4 — Session persistence:**
    1. After logging in, press F5 to refresh the page
    2. Expected: still on the home page, not redirected to /login

    **Test 5 — Protected routes accessible after login:**
    1. Navigate to http://localhost:3000/jobs
    2. Expected: jobs page loads (no redirect)
    3. Navigate to http://localhost:3000/cronjobs
    4. Expected: cronjobs page loads

    **Test 6 — Data isolation (if you have two users):**
    1. Create a second user: `pnpm create-user userb passb1234`
    2. As testuser, create a job (via the UI or POST /api/jobs)
    3. Log out (clear cookies manually in DevTools)
    4. Log in as userb
    5. Navigate to /jobs
    6. Expected: userb sees no jobs (testuser's job is not visible)

    **Test 7 — Logout:**
    1. While logged in, clear cookies manually in browser DevTools
    2. Navigate to /jobs
    3. Expected: redirected to /login
  </how-to-verify>
