# Jira Ticket Description Best Practices

Clear Jira tickets reduce misunderstandings and make development faster and easier.

---

# General Rules

### Be Clear

Use simple language. Describe exactly what needs to be done.

### Provide Context

Explain **why the work is needed**, not just **what to build or fix**.

### Make It Testable

Someone should be able to verify the work using only the ticket description.

### Define Acceptance Criteria

State clear conditions that define when the ticket is **done**.

### Keep It Short

Only include information necessary to understand and complete the task.

---

Below is a template of how feature ticket description should look like: When creating a feature ticket in jira, reference template below and try to fill it up as close as possible.

# Feature Ticket Template

```md
## Summary

Short description of the feature.

## Problem

Describe the problem this feature solves.

## Solution

High level explanation of how the feature should work.

## Scope

### In Scope

- Item
- Item

### Out of Scope

- Item
- Item

## User Story

As a **[user]**, I want **[capability]**, so that **[benefit]**.

## Acceptance Criteria

- [ ] Condition 1
- [ ] Condition 2
- [ ] Condition 3

## Notes

Optional technical notes if needed.
```
