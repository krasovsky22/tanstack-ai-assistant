Below is a template of how bug ticket description should look like: When creating a bug ticket in jira, reference template below and try to fill it up as close as possible.

```md
## Summary

Short description of the bug.

## Environment

- Environment: Production / Staging / Development
- Version:

## Steps to Reproduce

1. Step one
2. Step two
3. Step three

## Expected Result

Describe what should happen.

## Actual Result

Describe what currently happens.

## Impact

Describe the impact of the bug.
Examples:

- Blocks a user workflow
- Incorrect data
- UI issue

## Notes

Optional debugging information.
```
